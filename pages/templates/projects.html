Transcription Factor Enrichment Analysis (TFEA) is an open-source bioinformatics software developed at the BioFrontiers Institute, utilizing a computational method to analyze differential Transcription Factor activity. I contributed to the testing & bug finding of this software.

After installing and configuring the software on an HPC node, I created a web-accessible infrastructure on top of it that utilizes parallel computing power across its 24 cores. The web app allows researchers to generate entire TFEA pipelines by uploading their input files, and inputting some parameters in a simple, intuitive GUI. Users can also view and manage their past submissions, see job status, and view/download results directly in their browser. The backend leverages Celery to create dynamic, asynchronous job queues, and balance system load when handling multiple users' jobs at once. Users are notified of pipeline events via email alerts.

This dashboard was created to be a resource for local governments to consult in their decision making for closures and regulations during the COVID-19 pandemic. It uses anonymous Facebook user density data to track changes in population density over time, and aggregates it into interactive Plotly Dash apps for each city. With this tool, behaviors of both individual neighborhoods and the population at large can be inferred, such as how their movement and travel patterns have changed -- and if they hold -- after stay-at-home orders. The anonymized user data was collected from Facebook Data for Good. I contributed to debugging, and modifying the data ingestion of the original dashboard code.

I created a web app that integrates these dashboards, and separates multiple data sets into a different view for each city. The interactive dashboards support live updating, powered by a message-passing backend layer like Redis.

This internal tool visualizes collaboration between researchers, departments, and the core/support facilities at BioFrontiers (HPC/storage services, Advanced Light Microscopy, and Next-Generation Sequencing). It was created to identify patterns in researchers' collaborative habits with their publication co-authors, across their areas of research, their departments, and the funding tied to all of them; as well as compiling this data generated from users' queries into custom tables. In its bare-bones, early beta version, it visualizes simple relationships between researchers, their publications, their departments/affiliations, and the core services they use, into complex networks that mirror the structure and schema of the underlying graph database.

I created the concept and designed the infrastructure, choosing a graph database for the fastest performance while users are making many exploratory queries, and while executing path traversal algorithms. I built the Flask app to translate users' simple point-and-click interactions into complex Cypher queries, to generate an interactive network-like graph. The data was aggregated and processed by ETL pipelines I created in StreamSets, and written to a self-hosted Neo4j graph database instance.

Quantitation of Fluourescence Accumulation aftter DNA Damage (Q-FADD) is a piece of biophysics software that implements an analysis routine with free diffusion models to quantify the accumulation of fluorescenctly labeled proteins at the sites of DNA damage. The first of its pipelines analyzes movie files generated from fluourescence microscopy experiments, and processes them into files more appropriate for analytical model-fitting. The second pipeline uses these files to do the actual model-fitting computations in a grid-search routine. Both of these pipelines utilize parallel computing across a 24-core HPC node. I contributed to some of the testing & bug finding of this software.

After installing the software to a custom HPC node environment, I created a Django app that could directly leverage this software. The app allows researchers to generate both types of pipelines from a simple GUI that takes uploaded files and input parameters. The intuitive manager interface also lets users immediately use output files from the first pipeline as input files to the second one, with one click. Users can also manage jobs and monitor their status, as well as download their results directly in this interface. The backend runs Celery to create asynchronous job queues and workers configured specifically for each type of job, and to balance system load when handling multiple users' jobs at once. Users are notified of pipeline events and details via email alerts.

The BioFrontiers IT website serves a large spectrum of users and the resources they look for, including documentation on HPC services, remote storage and virtual machine hosting services, desktop and A/V support, news and events, workshops, and webinars. The site runs on a custom Wagtail solution designed to give the entire IT team the power to easily manage content, maintainance alerts, and events with just a few clicks.

I created the site from scratch, starting with a blank instance of the Python-based Wagtail CMS, and built custom models, content types, and frontend components specific to BIT's needs. These frontend and backend modules are dynamic, and allow team members to generate fully-featured, polished content in a matter of seconds. A custom user permissions solution integrated with LDAP also restricts access to certain content only viewable to researchers. A full search backend integrating tags, categories, and page keywords is powered with Elasticsearch.
